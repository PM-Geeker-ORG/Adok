# 产品的自我修养
作为产品应该经常反省自己的产品修养，不断学习和实践，成为真正的产品人。这里将会分享关于产品的笔记内容，可能无法做到非常体系也无法做到非常深刻，但会涉及到产品的多个方面，**希望大家在不足中可以遇见美好**。

> **关于New Bing（GPT-4）的对话**

Q：世界为什么需要产品经理？

A：产品经理是一个负责设计和管理产品的角色，他们需要了解用户的需求，分析市场的趋势，制定产品的目标和策略，协调开发和运营团队，不断优化和迭代产品，为用户提供更好的体验和价值。**```世界为什么需要产品经理呢？```** 因为产品经理可以**帮助创造更多有用、有趣、有影响力的产品，解决用户的痛点，满足用户的期望，推动社会的进步和创新**。

## 🔅 产品人的思维

> **```产品思维是产品最基础的东西，它具有抽象性，当一个产品人沉淀了自己的产品认知体系后，它又会变得现实性。但是由于变化才是永远不变的东西，所以产品思维也是一个动态的思维，产品人需要不断完善和创新，让产品越来越好```**

## 🔅 产品人的日常

> **```产品的日常工作属于多领域。如果在大团队，由于分工，可能会限定于几个固定的工作流；如果在小团队，可能就不再细分，让产品多方位执行。但是由于产品的定位是团队的核心，所以不断怎样，产品人的日常都需要在多领域中思考着工作。```**

| 工作名称 | 工作内容 |
| :---: | :--- |
| -- |  |
| **需求收集** | 收集来自外部和内部的产品需求情况并汇集到需求清单 |
| **需求评审** | 向团队相关成员进行需求文档的相关内容阐述和说明以便目标一致 |
| **需求分析** | 对收集后的需求进行合理思考分析和取舍并设置需求优先级 |
| **数据分析** | 运营中所收集到的数据进行适当的统计分析以便优化或创新 |
| **竞品分析** | 对目前主要竞争或龙头产品进行优势和劣势分析以便学习和创新 |
| **行业分析** | 对产品所在的行业进行调研并从中了解到行业的发展趋势和方向 |
| **商业分析** | 从经济效益上分析自身产品的商业模型和商业盈利以便改进 |
| -- |  |
| **版本管理** | 制定合理的产品发展路线并以版本形式进行管理和迭代更新 |
| **业务规划** | 随着市场或商业的变化需要不断规划自身产品的业务情况以作适应 |
| **沟通协调** | 对团队中不同部门的人员进行沟通和协作以致为达到工作效率 |
| **项目管理** | 规划项目全局情况并进行项目资源最优化的把控和按时交付 |
| **项目复盘** | 进行项目途中或结束的事实陈述并分析总结获得改进的收获 |
| -- |  |
| **文档撰写** | 撰写各种演示文档或需求文档以作为留档或供团队成员阅读 |
| **市场调研** | 收集和分析市场情况以便预测市场的方向为后续产品发展做准备 |
| **用户调研** | 对用户画像、喜好、习惯和反馈情况进行调研以便更亲近用户的心智 |
| **用户体验** | 能够站在用户的角度去体验用户的需求、情感、行为和反馈情况 |
| **产品设计** | 从真实的需求出发为产品的业务或流程设计出合理的使用价值 |
| **产品规划** | 基于产品现状和目标对产品制定良好的未来行动路径和计划方案 |
| **产品运营** | 对上线产品的运营维护以使产品得到良好运作和达到既定目标 |
| **产品优化** | 调整产品存在的问题或不足并持续执行优化设计和产品迭代 |
| **产品创新** | 为适应产品发展对原产品的功能进行创新或重新开发新款产品 |
| **产品验收** | 对待上线的产品进行验收测试确保产品流程达到闭环并可正常上线 |

**✅ 简单的产品工作流24步：**

1️⃣ ```Start:需求来源``` => **需求池记录** => **优先级评定** => **版本迭代需求确定** => **立项（可延后至评审之后）**=>

2️⃣ **原型设计** => **评审第一次** => **评审第二次** => **设计稿** =>

3️⃣ **设计稿评审** => **开发人员分工** => **开发排期** => **需求文档（可提前到评审之前）**=>

4️⃣ **数据库及开发接口文档评审** => **项目开发** => **产品开发沟通协助** => **测试用例** =>

5️⃣ **测试用例评审** => **测试环境搭建** => **提测** => **修复Bug** =>

6️⃣ **产品上线前验收** => **正式环境搭建** => **产品上线（可增加灰度上线）**=> **正式环境验收** => ```End:需求来源```

| :---: | :--- |
| -- |  |

| **买断模式** | 将所有成本直接转嫁到消费者身上，需要直接付费后才能使用产品服务 |

| **广告模式** | 网站免费或者以很低的价格提供内容给用户，换取用户浏览广告，由广告主付费 |
| **免费模式** | 提供免费服务可以获取大量用户，吸引到海量用户后便可以加速推广该产品到市场 |
| **增值模式** | 为客户提供的超出常规服务范围的服务，收费的服务同时会获取利润 |
| **门户模式** | 建设产品和服务信息的特定消费者社区，吸引不同类型消费者的注意力进入网络 |
| **社区模式** | 依赖于用户忠诚度、社交网络和用户创造内容UGC平台，搭建区域网络效应 |
| **O2O模式** | 将线下的商务机会与互联网结合，让线上互联网成为线下交易的平台 |
| **跨界模式** | 打破了原有的行业特征，高效率地整合几个行业来提升整体系统效率和优势 |
| -- | |
| **金融理财** | 将钱借给金融机构，金融机构用这笔钱去赚取更多利润，并以利息的方式返还利润 |

## 🔅 竞品分析

**✅ 三个基础问题**

1. ❗什么是竞品分析？

> **系统地对竞争对手的优势和劣势进行分析评价**

2. ❗竞品分析目的？

> **帮助自身产品实现市场功能；产品设计和可用性提供参考；提高自身产品的差异化**

3. ❗为什么做竞品分析？

> **为企业制定产品战略规划；了解竞争对手的产品和市场动态；掌握竞争对手的资本背景和产品运营策略；快速自我调整并提供市场占有率；形成有效的系统化思想和客观准确方向等**

**✅ 竞品分析怎么做**

1. **三个方面**

描述性（说明我们和竞争对手有哪些差距）

定性分析（说明我们和竞品的差距在哪里） 

定量分析（说明我们和竞争对手的差距有多大||功能价值=功能得分*级别得分*权重值）

2. **分析维度**

战略定位BRD；商业定位MRD；用户群体MRD；产品功能PRD；产品界面（交互方式和视觉）PRD

3. **竞品选择**

直接竞争者；间接竞争者；潜在竞争者

4. **分析方法**

SWOT分析（优势、劣势、机会、威胁）四大板块定量评分表格；

竞争分析模型SAPPEALS：价格 可获得性 包装 性能 易用性 保证 生命周期 社会接受度

5. **获取渠道**

从内部市场、运营部门、管理层等收集信息；

行业媒体平台新闻及论坛；

建立持续的产品市场信息小组；

调查核心用户、活跃用户、普通用户不同需求；

竞争对手网站、交流互动平台、促销活动；

季表和年表财报；各大人才招聘网站信息和招聘更新；

通过谷歌搜索了解国外同行业信息和盈利模式；

6. **获取内容**

公司技术、产品、运营团队规模及核心目标和行业品牌影响力；

实际季度年度盈利和公司主盈利的产品线；

用户群体覆盖面和市场占有率；

产品功能稳定性、易用性、用户交互和技术框架细分及对比；

产品平台及官方的排名和关键字及外链数量
