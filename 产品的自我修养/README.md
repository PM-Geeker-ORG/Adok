# 产品的自我修养
作为产品应该经常反省自己的产品修养，不断学习和实践，成为真正的产品人。这里将会分享关于产品的笔记内容，可能无法做到非常体系也无法做到非常深刻，但会涉及到产品的多个方面，希望大家在不足中可以遇见美好。

## 🔅 产品人的思维

> **```产品思维是产品最基础的东西，它具有抽象性，当一个产品人沉淀了自己的产品认知体系后，它又会变得现实性。但是由于变化才是永远不变的东西，所以产品思维也是一个动态的思维，产品人需要不断完善和创新，让产品越来越好```**

**✅ 笔记内容-简单语句**

1. ⚡放下自己：先考虑别人的感受，你可能更熟悉另一个说法，即同理心

2. ⚡不抱怨不放弃，思考解决方案
3. ⚡主动担责，不推卸不逃避：先想自己的问题，哪怕别人是主因
4. ⚡执行高于计划：你还在等什么，谁开始跑步的时候都摔跤
5. ⚡放空自己：你认为别人应该知道的 ≠ 别人知道的
6. ⚡挖的深看得真：真实的诉求像女孩子的爱情，不会轻易得到，但绝对是值得挖掘的宝藏
7. ⚡不做传话筒：说的不等于要的，不要听一面之词
8. ⚡不急于做决定：努力看清事情全貌后在判断
9. ⚡先谈价值再谈利益：要为团队或公司开发一条利益增长的曲线，但在这之前先谈谈你创造了什么价值
10. ⚡你只是其中一员：做个Team Member，而不是CPU
11. ⚡质疑一切挑战自己：拒绝把存在即合理当座右铭，把Challenge当日常

**✅ 追加一些公开网络中获得的语句**

> - 产品的首要问题应该是产品的定位问题
> - 不要只盯着产品本身，更要从生态和系统的角度来思考
> - 好的产品体验，就是“最简单、最直观”的体验
> - 要到具体的场景中去设计和检验产品
> - 善于从用户的行为而不仅是语言中获得产品反馈
> - 无论什么产品，都要开始重视品味和艺术
> - 要让产品自身有引发喜爱和传播的能力
> - 眼里不要只有“产品功能”，更要多多考虑“人”
> - 极致的产品思维，就是用产品思考一切
> - 不要忘记初心，去做自己都不用、甚至讨厌的产品
> - 要形成产品文化，CEO要做最大的产品经理

## 🔅 产品人的日常

> **```产品的日常工作属于多领域。如果在大团队，由于分工，可能会限定于几个固定的工作流；如果在小团队，可能就不再细分，让产品多方位执行。但是由于产品的定位是团队的核心，所以不断怎样，产品人的日常都需要在多领域中思考着工作。```**

| 工作名称（无序） | 工作内容 |
| :---: | :--- |
| 需求收集 | 收集来自外部和内部的产品需求情况并汇集到需求清单 |
| 需求评审 | 向团队相关成员进行需求文档的相关内容阐述和说明以便目标一致 |
| 需求分析 | 对收集后的需求进行合理思考分析和取舍并设置需求优先级 |
| 数据分析 | 运营中所收集到的数据进行适当的统计分析以便优化或创新 |
| 竞品分析 | 对目前主要竞争或龙头产品进行优势和劣势分析以便学习和创新 |
| 行业分析 | 对产品所在的行业进行调研并从中了解到行业的发展趋势和方向 |
| 商业分析 | 从经济效益上分析自身产品的商业模型和商业盈利以便改进 |
| -- |  |
| 版本管理 | 制定合理的产品发展路线并以版本形式进行管理和迭代更新 |
| 业务规划 | 随着市场或商业的变化需要不断规划自身产品的业务情况以作适应 |
| 沟通协调 | 对团队中不同部门的人员进行沟通和协作以致为达到工作效率 |
| 项目管理 | 规划项目全局情况并进行项目资源最优化的把控和按时交付 |
| 项目复盘 | 进行项目途中或结束的事实陈述并分析总结获得改进的收获 |
| -- |  |
| 文档撰写 | 撰写各种演示文档或需求文档以作为留档或供团队成员阅读 |
| 市场调研 | 收集和分析市场情况以便预测市场的方向为后续产品发展做准备 |
| 用户调研 | 对用户画像、喜好、习惯和反馈情况进行调研以便更亲近用户的心智 |
|     用户体验     | 能够站在用户的角度去体验用户的需求、情感、行为和反馈情况     |
| 产品设计 | 从真实的需求出发为产品的业务或流程设计出合理的使用价值 |
| 产品规划 | 基于产品现状和目标对产品制定良好的未来行动路径和计划方案 |
| 产品运营 | 对上线产品的运营维护以使产品得到良好运作和达到既定目标 |
| 产品优化 | 调整产品存在的问题或不足并持续执行优化设计和产品迭代 |
| 产品创新 | 为适应产品发展对原产品的功能进行创新或重新开发新款产品 |
| 产品验收 | 对待上线的产品进行验收测试确保产品流程达到闭环并可正常上线 |

**✅ 简单的产品工作流24步：**

```Start:需求来源``` =》需求池记录 =》优先级评定 =》版本迭代需求确定 =》 立项（可延后至评审之后）=》

原型设计 =》评审第一次 =》评审第二次 =》设计稿 =》

设计稿评审 =》开发人员分工 =》开发排期 =》需求文档（可提前到评审之前）=》

数据库及开发接口文档评审 =》项目开发 =》产品开发沟通协助 =》测试用例 =》

测试用例评审 =》测试环境搭建 =》提测 =》修复Bug =》

产品上线前验收 =》正式环境搭建 =》产品上线（可增加灰度上线）=》正式环境验收 =》```End:需求来源```

## 🔅 互联网思维

> **```接着上面的商业模式画布再谈一下这个精炼而实用的商业思维。```**

1.	用户思维：关注经营理念和消费者，用户体验至上，大众化，粉丝化，参与感
2.	简约思维：关注品牌和产品规划，专注，少则是多，简约既是美
3.	极致思维：关于产品和服务体验，让用户尖叫的产品，服务即营销
4.	迭代思维：关于创新流程，微创新，快速迭代，适应用户需求
5.	流量思维：关于业务运营，免费是更好的收费，坚持到质变的临界点
6.	社会化思维：关于传播链和关键链，口碑营销，开发大众协作
7.	大数据思维：关于企业资产和核心竞争力，数据分析用户，大数据驱动运营管理
8.	平台思维：关于商业模式和组织形态，多方共赢平台生态圈，善用现有平台
9.	跨界思维：关于产业边界和创新，敢于自我颠覆，携用户以令诸侯，打破利益分配格局
10.	商业模式画布：用来描述商业模式和评估商业模式以及改变商业模式的通用语言
